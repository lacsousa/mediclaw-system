O Problema Motivação para o MediClaw

O que é o MediClaw?

Plataforma web full-stack de apoio à decisão clínica (CDSS) que combina quatro pilares tecnológicos:

Django REST Framework

Backend API robusto com autenticação JWT, rate limiting e envelope padronizado

LLMs Configuráveis

OpenAI GPT-4o-mini ou Google Gemini, alternáveis via variável de ambiente

RAG com ChromaDB

Respostas embasadas em literatura científica via busca semântica vetorial

Next.js Frontend

Interface interativa com streaming SSE em tempo real via EventSource

Restrição Crítica

Ética Médica

A IA nunca emite diagnóstico médico ou prescrição.

  • Guardrails determinísticos: filtros pré e pós LLM bloqueiam diagnóstico e prescrição
  • Disclaimer obrigatório: injetado automaticamente em toda resposta clínica
  • LGPD Art. 11: dados de saúde tratados como dados pessoais sensíveis

Pipeline de IA — 7 Etapas

1

Guardrail de Entrada

Filtros keyword/regex bloqueiam diagnóstico, prescrição, urgência e gibberish — custo zero, sem chamar o LLM

2

Captura de Dados — Skills

health_summary (7d), calculate_bmi, convert_units enriquecem o contexto automaticamente

3

Busca RAG

Embedding com text-embedding-3-small → top-5 chunks, score ≥ 0.75 em ChromaDB

4

Montagem do Prompt

SYSTEM_PROMPT com contexto RAG + dados de saúde + últimas 6 mensagens + pergunta do usuário

5

Chamada ao LLM

Modo síncrono (POST) ou Streaming SSE (GET) — tokens entregues token a token via EventSource

6

Guardrail de Saída

Intercepta diagnóstico/prescrição na resposta do LLM e injeta disclaimer médico obrigatório

7

Persistência + Disclaimer

Persiste Message com tokens_used, metadata (citações, latência) e registra no audit log sem PII

Endpoints de IA — 4 implementados (mínimo: 2)

Endpoint Tipo Pipeline
POST /api/v1/conversations/{id}/messages/ Síncrono Guardrail → Skills → RAG → LLM → Guardrail saída
GET /api/v1/conversations/{id}/stream/ Streaming SSE Mesmo pipeline, token a token via EventSource
POST /api/v1/admin/knowledge/upload/ Ingestão RAG Embeddings + indexação ChromaDB (PDF/TXT/MD ≤ 10MB)
GET /api/v1/health/summary/ Skill IA Agregação biométrica 7d + análise por IA

* Todos os endpoints com autenticação JWT Bearer + rate limiting DRF

Segurança

Tokens JWT

  • Access Token: 30 minutos — limita exposição
  • Refresh Token: 1 dia — renova sem novo login
  • Stateless: sem Redis ou sessão em servidor
  • Header: Authorization: Bearer <token>

Rate Limiting + Outras Camadas

  • Anônimo: 30/min · Usuário: 60/min · Chat: 10/min
  • Multi-tenancy: médicos isolados por doctor=request.user
  • CORS: origens explícitas, sem *
  • Sem PII em logs: conteúdo de mensagens filtrado
